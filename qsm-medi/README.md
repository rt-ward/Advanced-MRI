# QSM-MEDI Pipeline

## Introduction

This repository stores the files to build the invivo QSM pipeline for Rush
data. The pipeline is based on MEDI toolbox and has slight option changes
to yield better quality QSM maps. Currently supports in-vivo scans from UC
and Rush (Siemens). QSM MRI protocol changes will require review of this
QSM pipeline. 16 GB of RAM and 3.5 cores at most havebeen observed to be
used during runtime. Docker image stored on: mriinvivo.azurecr.io/invivoqsm:1.2

## Pipeline Description

Frequency maps are generated by non-linearly fitting all the echoes from the
gradient-echo sequence in the complex plane. Background field is removed using
Projection onto Dipole Fields (PDF). Dipole inversion with reference value
is performed with MEDI+0. QSM maps have a reference value set to the QSM value
of the CSF. Note that the background voxels have a value of the reference
value.

## Pipeline Dependencies

- dcm2niix pipeline from data management
  - requires nifti images and json files containing central frequency info
- HD-BET pipeline
  - requires brain mask (could be generated with other methods, as long as accurate)

## Running Docker Container

Requirements

- Input Folder = Folder of QSM nifti and json files (can contain other files)
- Mask Folder = Folder containing brain mask
- Output folder

Mount points

- Input Folder to /input
- Mask Folder to /mask
- Output Folder to /output

Input arguments

- Processing code. Must be either
  - uc
  - siemens
  - markvcid_uc
- Filename of mask
  
Example
`docker run -v /work/160621_00_75458387/75458387_00_nii:/input/ -v
  /qsm_folder:/output -v /work/hdbet_masks:/mask -it
  mriinvivo.azurecr.io/invivoqsm:1.2 siemens
  mask-hdbet-160621_00_75458387.nii.gz`

## Output Files

### Main Files

- QSM.nii.gz = magnetic susceptibility map in ppm units
- Mask_QSM.nii.gz = mask for QSM map (after erosion of input mask)

### Intermediate Files

- iMag.nii.gz = magnitude image calculated as the root mean squared of
  magnitude images across echoes
- iFreq_raw.nii.gz = wrapped magnetic field map
- iFreq_raw_N_std.nii.gz = a priori error estimate from field calculation
- iFreq_raw_p0.nii.gz = frequency offset map from field calculation
- iFreq_raw_relres.nii.gz = relative residuals from field calculation
- iFreq_laplacian.nii.gz = unwrapped magnetic field map (laplacian)
- Mask_CSF.nii.gz = CSF mask derived from R2s processing within MEDI toolbox
- Mask_preQSM.nii.gz = input mask
- R2s.nii.gz = R2s map with MEDI toolbox method
- RDF.mat = local field map as a matlab file
- RDF.nii.gz = local field map
- results (folder) = MEDI_L1 output as MATLAB matrices

- No longer used:
  - iFreq_spatial.nii.gz = unwrapped magnetic field map (region-growing)

## Optimization

The following changes were made to the default MEDI pipeline:

- 'a' value related to MERIT option was lowered to 4 from 6 (noisier QSM data
  -> lower a)
  - after each iteration, voxels with absolute demeaned residual greater than
    a*stdev(residuals) are marked as 'bad' voxels and have less weight in
    future iterations
- In MEDI_L1.m, cg_max_iter=300 instead of 100
- PDF method has max iterations=300 instead of 30
- To lower memory usage, cleared variables when the variable is no longer
  needed for processing
- Added nifti saving of intermediate and output files
- Calculated b0 direction using method similar to that in the read dicom
  function

Summary of important parameters: 3a for merit, 5 smv, 300iter

## Notes

- Using all echoes from QSM data yielded better QSM maps than that from the
  last echo being excluded
- Lowering 'a' value too low can make QSM image look better with less streaking
  but overly smooth
- The default 30 PDF iterations does not seem enough to remove the background
  field
- SMV option in MEDI_L1 makes a significant improvement in QSM map but at the
  cost of losing 5 mm at the edges of the brain
  - The loss of edges may be okay since the edges of the QSM map without SMV
    are not reliable. SMV improves QSM appearance throughout
  - Lower SMV value yields lesser quality QSM maps
- Laplacian unwrap resolves phase wraps created from blood vessels that cannot
  be resolved from region growing unwrap
- Observed that QSM values of blood vessels seemed more accurate with region
  growing and are suppressed with Laplacian (blood vessels should have sharp
  contrast)
- Decided to use Laplacian to handle open ended fringe lines observed in
  latest mg acquisition folder in 2021
- Considered using zero padding but the sizes of the images caused memory
  usage to go above 16 GB.
- Phase correction was modified to include the phase encoding direction.
  Did not include phase encoding direction, by default

## Build

1. Download build folder (contains modified MEDI_toolbox, niftiIO, and scripts)
   from cloud ()
2. Make changes/optimizations to MEDI code
3. Using MATLAB Compiler, compile process_QSM.m as main file with output to
   build/matlab_compiler
   - Save as new folder
4. Run create_docker.sh
   - Modify the version of matlab compiler runtime in docker image to match
     matlab version used to compile, if needed

## Adding new processing codes

When protocol changes are made, a new processing code may need to be added.
To do this, follow the Build steps, and at step 2, add another if statement
for the new processing code.

libncurses5 is needed for MCR, but is not available in ubuntu 24.04
